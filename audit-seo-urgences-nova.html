Problèmes détectés

  • /macon-urgence : title trop court (43 car.) — manque « 24h/24 » et « en moins de 2h » pour la cohérence
  • Tous les titles sont identiques en structure — risque de cannibalisation perçue si Google les agrège
  • Manque de mots-clés géolocalisés (ex: « Paris », « Île-de-France »)

Recommandation Title

Ajouter une variation géographique et différencier chaque title :

  • Serrurier urgence 24h/24 Paris | Nova — Intervention < 2h, séquestre
  • Plombier urgence 24h/24 | Fuite d'eau ? Nova intervient en < 2h
  • Maçon urgence 24h/24 | Nova — Fissure, effondrement, intervention rapide

CRITIQUE : Contenu non crawlable

Le composant TradeUrgency est marqué "use client". Tout le contenu SEO (H1, texte, exemples, FAQ) est invisible pour Google.

Solution : Extraire la partie statique dans un Server Component et ne garder "use client" que pour les interactions (modal, boutons).

Contenu manquant pour le SEO

  • Absent Section FAQ — Les FAQs existent dans trades.ts mais ne sont PAS affichées sur les pages urgence (uniquement sur les pages métier standard)
  • Absent Contenu éditorial — Pas de paragraphes explicatifs (comment ça marche, tarifs indicatifs, zones d'intervention)
  • Absent Section « Comment ça marche » — Le stepper d'escrow est visuel mais pas textuel
  • Absent Témoignages / Avis — Pas de social proof textuelle crawlable
  • Absent Zone d'intervention — Aucune mention de villes/régions couvertes

Recommandation Structure

Ajouter sur chaque page urgence :

  1. Section FAQ (3-5 questions) avec balisage FAQPage Schema.org
  2. Section « Comment fonctionne l'urgence Nova » (300+ mots, H2)
  3. Section « Tarifs indicatifs » (H2 + tableau)
  4. Section « Zones d'intervention » (H2 + liste de villes)
  5. Breadcrumb structuré avec Schema.org BreadcrumbList

11. Plan de modifications proposées

Modification 1 — Architecture SSR/Client hybride P0

Fichier : src/components/features/trade-urgency.tsx

Problème : Tout le composant est "use client" → contenu invisible pour Google

Solution :

  1. Créer trade-urgency-server.tsx (Server Component) pour le contenu statique : H1, textes, FAQ, breadcrumb, sections éditoriales
  2. Garder trade-urgency-client.tsx (Client Component) uniquement pour le modal et les boutons interactifs
  3. Composer les deux dans page.tsx

Impact estimé : +8 points SEO (mots-clés + structure crawlables)

Modification 2 — Ajouter les données structurées Schema.org P0

Fichiers : Chaque layout.tsx de page urgence

Solution : Ajouter un composant JsonLd avec :

  • FAQPage : injecter les 5 FAQ de trades.ts
  • Service : décrire le service d'urgence (provider: Nova, areaServed: France, etc.)
  • BreadcrumbList : Accueil > {Métier} > Urgence
  • HowTo : Les 4 étapes de l'escrow

Exemple de structure :

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "Combien coûte une ouverture de porte ?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "Le prix dépend de la serrure... entre 90 et 180 EUR."
      }
    }
  ]
}
</script>

Impact estimé : +4 points (featured snippets)

Modification 3 — Créer robots.ts P0

Fichier à créer : src/app/robots.ts

import { MetadataRoute } from "next";

export default function robots(): MetadataRoute.Robots {
  return {
    rules: { userAgent: "*", allow: "/", disallow: ["/api/", "/admin/"] },
    sitemap: "https://nova.fr/sitemap.xml",
  };
}

Modification 4 — Compléter le sitemap P0

Fichier : src/app/sitemap.ts

Ajouter : les 8 pages métier + 5 pages urgence + page /comment-ca-marche

Modification 7 — Ajouter contenu éditorial SEO P1

Pour chaque page urgence, ajouter 2 sections textuelles :

  1. « Comment fonctionne l'intervention d'urgence Nova » (H2)

  2. « Tarifs indicatifs {trade.name} urgence » (H2)
    Tableau avec fourchettes de prix par type d'intervention. Déjà présent dans les FAQ mais pas structuré en tableau.
